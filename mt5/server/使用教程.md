# MT5交易服务器使用教程

## 📋 文档结构
1. 启动MT5，登录一个账户，并开启算法交易功能
2. 设置自定义时间段和默认仓位大小
3. 启动服务
4. 两种接受信号方式：webhook和本地chrome插件调用
5. 设置TradingView的警报，格式有要求

---

## 1. 启动MT5并开启算法交易

### 1.1 启动MT5客户端
1. **打开MetaTrader 5应用程序**
   - 双击桌面图标或从开始菜单启动
   - 等待程序完全加载

2. **登录交易账户**
   - 输入您的账户号码、密码和服务器
   - 点击"登录"按钮
   - 确认连接成功（右下角显示绿色信号）

### 1.2 开启算法交易功能
1. **启用自动交易**
   - 在工具栏找到"自动交易"按钮（通常是一个绿色的图标）
   - 点击该按钮，确保它呈激活状态（绿色高亮）
   - 如果按钮是灰色的，说明自动交易被禁用

2. **验证设置**
   - 在"工具" → "选项" → "专家顾问"中
   - 确保"允许自动交易"被勾选
   - 确保"允许DLL导入"被勾选（如果需要）

### 1.3 检查账户状态
```
账户信息应显示：
- 账户号码：正确的登录号
- 余额：当前账户余额
- 净值：当前账户净值
- 连接状态：已连接
```

## 2. 设置自定义时间段和默认仓位大小

### 2.1 配置自定义时间段

编辑 `config.yaml` 文件中的时间段设置：

```yaml
# 自定义时间段配置
custom_intervals:
  interval1:
    name: "欧洲时段"
    start_time: "08:00"
    end_time: "16:00"
    timezone: "Europe/London"
    description: "伦敦交易时间"

  interval2:
    name: "美洲时段"
    start_time: "14:00"
    end_time: "22:00"
    timezone: "America/New_York"
    description: "纽约交易时间"

  interval3:
    name: "亚洲时段"
    start_time: "01:00"
    end_time: "09:00"
    timezone: "GMT+8"
    description: "亚洲交易时间"

  interval4:
    name: "自定义时段"
    start_time: "13:00"
    end_time: "15:00"
    timezone: "GMT+8"
    description: "自定义交易时间"
```

### 2.2 配置默认仓位大小

在 `config.yaml` 文件中设置交易参数：

```yaml
# 交易配置
trading:
  default_volume: 0.1      # 默认手数
  max_volume: 10.0         # 最大手数
  min_volume: 0.01         # 最小手数
  max_slippage: 3          # 最大滑点
  magic_number: 12345      # 魔术数字
  allowed_symbols: []      # 允许的交易品种（空=全部）
```

### 2.3 时区格式支持

系统支持多种时区格式：
- **标准时区**: `Europe/London`, `America/New_York`, `Asia/Shanghai`
- **GMT格式**: `GMT+8`, `GMT-5`, `UTC+0`

## 3. 启动服务

### 3.1 启动命令

在命令行中执行：

```bash
cd mt5/server
python app.py
```

### 3.2 成功启动标志

看到以下信息表示启动成功：

```
MT5: CONNECTED - Account: 67424067
Trading hours manager initialized with 4 custom intervals
Server running on http://127.0.0.1:5000
```

### 3.3 开启时间区间

服务器启动后，执行以下命令开启交易时间：

```bash
curl -X POST http://127.0.0.1:5000/webhook \
  -H "Content-Type: text/plain; charset=utf-8" \
  -d "开启时间区间"
```

**成功响应**：
```json
{
  "success": true,
  "message": "Trading hours enabled for all custom intervals"
}
```

### 3.4 验证服务状态

```bash
curl http://127.0.0.1:5000/health
```

**正常响应示例**：
```json
{
  "status": "healthy",
  "mt5_connected": true,
  "account_info": {
    "login": 67424067,
    "balance": 99699.19,
    "equity": 99699.19
  },
  "trading_hours": {
    "enabled": true,
    "trading_allowed": true
  }
}
```

## 4. 两种接受信号方式

### 4.1 Webhook方式（推荐）

**优势**：
- 实时性强
- 配置简单
- 支持所有TradingView功能

**配置方法**：
1. 在TradingView警报中设置Webhook URL: `http://127.0.0.1:5000/webhook`
2. 选择POST方法
3. 设置消息格式（见第5节）

**测试命令**：
```bash
curl -X POST http://127.0.0.1:5000/webhook \
  -H "Content-Type: text/plain; charset=utf-8" \
  -d "开多 XAUUSD 仓位=0.01"
```

### 4.2 本地Chrome插件调用

**优势**：
- 无需公网IP
- 安全性高
- 适合本地开发

**配置方法**：
1. 安装Chrome插件（如果有）
2. 配置插件连接到本地服务器
3. 设置插件监听TradingView页面

**注意**：此功能需要额外的插件支持，具体实现方式可能因插件而异。

## 5. 设置TradingView警报格式

### 5.1 基本交易命令格式

#### 开仓操作
```
开多 {{ticker}} 仓位=0.05
开空 {{ticker}} 仓位=0.1
```

#### 平仓操作
```
平多 {{ticker}}
平空 {{ticker}}
全平 {{ticker}}
```

#### 带止损止盈
```
开多 {{ticker}} 仓位=0.05 止损=3350 止盈=3370
开空 {{ticker}} 仓位=0.1 止损=1.1050 止盈=1.0950
```

### 5.2 Pine Script示例

```pinescript
//@version=5
strategy("MT5 Webhook Strategy", overlay=true)

// 策略参数
volume = input.float(0.1, "Position Size")
sl_percent = input.float(1.0, "Stop Loss %")
tp_percent = input.float(2.0, "Take Profit %")

// 交易信号
ma20 = ta.sma(close, 20)
ma50 = ta.sma(close, 50)

buy_signal = ta.crossover(ma20, ma50)
sell_signal = ta.crossunder(ma20, ma50)

// 执行交易并发送警报
if buy_signal
    strategy.entry("Long", strategy.long)
    alert("开多 " + syminfo.ticker + " 仓位=" + str.tostring(volume), alert.freq_once_per_bar)

if sell_signal
    strategy.close("Long")
    alert("平多 " + syminfo.ticker, alert.freq_once_per_bar)
```

### 5.3 TradingView警报设置

1. **创建警报**
   - 在图表上右键选择"添加警报"
   - 或点击顶部工具栏的警报图标

2. **配置条件**
   - 选择您的策略或指标
   - 设置触发条件

3. **设置Webhook**
   - 在"通知"选项卡中勾选"Webhook"
   - URL: `http://127.0.0.1:5000/webhook`
   - 消息格式：使用上述命令格式

4. **测试警报**
   - 保存警报后，可以手动触发测试
   - 检查服务器日志确认接收到信号

### 5.4 JSON格式（高级）

如果需要使用JSON格式：

```json
{
  "action": "buy",
  "symbol": "{{ticker}}",
  "volume": 0.05,
  "sl": 3350,
  "tp": 3370,
  "comment": "TradingView Signal"
}
```

## ❌ 常见问题解决

### 问题1: 服务器启动失败
**可能原因**: 端口被占用、Python环境问题
**解决方案**:
```bash
# 检查端口占用
netstat -ano | findstr :5000
# 结束占用进程或更改端口
```

### 问题2: MT5连接失败
**可能原因**: MT5未启动、未登录、网络问题
**解决方案**: 确保MT5客户端已登录并连接正常

### 问题3: 交易时间限制
**错误**: `Trading not allowed at current time`
**解决方案**: 执行 `开启时间区间` 命令

### 问题4: 自动交易禁用
**错误**: `AutoTrading disabled by client`
**解决方案**: 在MT5客户端启用自动交易按钮
